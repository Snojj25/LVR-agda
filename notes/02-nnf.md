## 1. What is Negation Normal Form?

A propositional formula is in **Negation Normal Form** (NNF) when negation
appears *only* directly in front of a propositional variable. No
`¬¨(a ‚àß b)`, no `¬¨(a ‚à® b)`, no `¬¨¬¨œÜ`.

If the abstract syntax of formulas is

```
Formula  œÜ ::=  Var n  |  ¬¨ œÜ  |  œÜ ‚àß œÜ  |  œÜ ‚à® œÜ
```

then NNF is the sub-grammar generated by

```
Literal  ‚Ñì ::=  Var n  |  ¬¨ Var n
NNF      œà ::=  ‚Ñì  |  œà ‚àß œà  |  œà ‚à® œà
```

The crucial restriction: at the NNF level there is **no negation
constructor**. The only place `¬¨` can occur is hidden inside a literal,
glued to a variable.

### Why is this called a "normal form"?

A "normal form" means two things at once:

1. **Existence.** Every formula `œÜ` is logically equivalent to some
   formula in the normal form.
2. **Structural simplicity.** The normal-form grammar is *more
   restrictive* than the full grammar, so structural induction has fewer
   cases.

`to-nnf` witnesses existence. Structural simplicity is the missing
`¬¨`-constructor ‚Äî functions over `NNF` have three cases (`lit`, `_‚àßn_`,
`_‚à®n_`) instead of four.

This pays off later: Tseytin only handles three node shapes; SAT-style
algorithms over NNF need not recompute polarity; inductive proofs avoid
the `¬¨œà` branch.

---

## 2. Why bother ‚Äî what is NNF good for?

- **Launching pad for CNF and DNF.** Both are sub-grammars of NNF,
  further restricting how `‚àß` and `‚à®` nest. The pipeline
  `Formula ‚Üí NNF ‚Üí CNF/DNF` starts with this note.
- **Tseytin needs NNF as input.** The Tseytin transformation (Problem 10,
  note 06) walks an `NNF` introducing a fresh variable per internal node,
  with no polarity reasoning needed.
- **Cleaner semantic arguments.** Monotonicity-style arguments ("more
  true variable ‚áí more true formula") fail on `Formula` because of `¬¨`,
  but hold on NNF (with the right generalisation to literals).
- **Polarity is explicit.** Each variable occurrence carries `pos n` or
  `neg n`. For algorithms that count positive vs negative occurrences
  (DPLL, pure-literal elimination), NNF is natural.

## 3. The `Literal` datatype

In `src/Solution.agda` (lines 58‚Äì60):

```agda
data Literal : Set where
  pos : ‚Ñï ‚Üí Literal     -- the literal  Var n
  neg : ‚Ñï ‚Üí Literal     -- the literal  ¬¨ Var n
```

A `Literal` is a tagged variable index. `pos n` represents `Var n`;
`neg n` represents `¬¨ Var n`. Two things to notice:

1. **Negation is "frozen into" the constructor.** No constructor takes
   another `Literal` and negates it ‚Äî to negate, you pattern-match and
   rebuild.
2. **The invariant "¬¨ wraps a variable only" is structural.** With only
   two constructors, each naming a single `‚Ñï`, it is impossible to
   construct a value that violates the invariant. We are not asserting
   correctness via an external predicate ‚Äî the type makes bad values
   nonexistent.

A recurring theme: **prefer making bad states unrepresentable to ruling
them out after the fact.** A predicate-style `IsNNF : Formula ‚Üí Set`
would also work, but every consumer would carry an `IsNNF œÜ` proof and
pattern-match on it. The `Literal` / `NNF` design pushes the invariant
into the data shape.

A small helper used later (lines 303‚Äì305):

```agda
flip : Literal ‚Üí Literal
flip (pos n) = neg n
flip (neg n) = pos n
```

`flip ‚Ñì` is the literal denoting the negation of `‚Ñì`, used in Tseytin.

## 4. The `NNF` datatype

Lines 62‚Äì65:

```agda
data NNF : Set where
  lit  : Literal ‚Üí NNF
  _‚àßn_ : NNF ‚Üí NNF ‚Üí NNF
  _‚à®n_ : NNF ‚Üí NNF ‚Üí NNF
```

The mirror of `Formula`, with two changes:

| `Formula`           | `NNF`               | comment                                  |
|---------------------|---------------------|------------------------------------------|
| `var : ‚Ñï ‚Üí Formula` | `lit : Literal ‚Üí NNF` | leaves are literals, not bare variables |
| `¬¨f_ : Formula ‚Üí Formula` | *(absent)*          | no negation constructor at all          |
| `_‚àßf_`              | `_‚àßn_`              | renamed to keep the two grammars apart  |
| `_‚à®f_`              | `_‚à®n_`              | likewise                                |

The missing `¬¨f_` is the only structural difference; leaves are richer
(carrying polarity), and one of four cases is dropped.

The `n` suffix distinguishes NNF connectives from `_‚àßf_`/`_‚à®f_` on
`Formula` and `_and_`/`_or_` on `Bool` ‚Äî three notions of and/or coexist
in the module.

## 5. The conversion algorithm: pushing `¬¨` to the leaves

The textbook algorithm rewrites `œÜ` until every negation sits on a
variable, using:

```
¬¨(a ‚àß b)   ‚â°   ¬¨a ‚à® ¬¨b      -- De Morgan (and)
¬¨(a ‚à® b)   ‚â°   ¬¨a ‚àß ¬¨b      -- De Morgan (or)
¬¨¬¨ œÜ       ‚â°   œÜ            -- double negation elimination
¬¨(Var n)   ‚â°   neg n        -- base case: produce a literal
```

Think of `¬¨` as a token wanting to reach a variable. At `‚àß` or `‚à®`, De
Morgan dives it past, flipping the connective. At another `¬¨`, the two
cancel. At `Var n`, it is absorbed into `neg n`.

Non-negated `‚àß`, `‚à®`, `Var` just recurse.

The *specification* is "iterate the rules to a fixed point". The
*implementation* problem: encode this as a structurally recursive Agda
function the termination checker accepts.

## 6. The two-function trick

The natural first attempt:

```agda
to-nnf : Formula ‚Üí NNF
to-nnf (var n)            = lit (pos n)
to-nnf (¬¨f var n)         = lit (neg n)
to-nnf (¬¨f ¬¨f œÜ)          = to-nnf œÜ
to-nnf (¬¨f (a ‚àßf b))      = to-nnf (¬¨f a ‚à®f ¬¨f b)   -- De Morgan
to-nnf (¬¨f (a ‚à®f b))      = to-nnf (¬¨f a ‚àßf ¬¨f b)   -- De Morgan
to-nnf (a ‚àßf b)           = to-nnf a ‚àßn to-nnf b
to-nnf (a ‚à®f b)           = to-nnf a ‚à®n to-nnf b
```

Semantically correct, but it fails Agda's termination check.

In the De Morgan case:

```agda
to-nnf (¬¨f (a ‚àßf b)) = to-nnf (¬¨f a ‚à®f ¬¨f b)
```

The recursive argument `¬¨f a ‚à®f ¬¨f b` is **not** a strict structural
sub-term of `¬¨f (a ‚àßf b)` ‚Äî we kept `a` and `b` but wrapped them in fresh
`¬¨f` and changed the outer connective. By constructor count it is no
smaller. Agda gives up.

(It really isn't smaller in any easy structural sense ‚Äî proving
termination by hand needs a custom well-founded measure, doable but
painful.)

### The fix

Define **two** functions. Lines 85‚Äì96:

```agda
nnf‚Å∫ : Formula ‚Üí NNF
nnf‚Åª : Formula ‚Üí NNF

nnf‚Å∫ (var n)   = lit (pos n)
nnf‚Å∫ (¬¨f œÜ)    = nnf‚Åª œÜ
nnf‚Å∫ (a ‚àßf b)  = nnf‚Å∫ a ‚àßn nnf‚Å∫ b
nnf‚Å∫ (a ‚à®f b)  = nnf‚Å∫ a ‚à®n nnf‚Å∫ b

nnf‚Åª (var n)   = lit (neg n)
nnf‚Åª (¬¨f œÜ)    = nnf‚Å∫ œÜ                 -- ¬¨¬¨œÜ ‚â° œÜ
nnf‚Åª (a ‚àßf b)  = nnf‚Åª a ‚à®n nnf‚Åª b       -- ¬¨(a‚àßb) ‚â° ¬¨a ‚à® ¬¨b
nnf‚Åª (a ‚à®f b)  = nnf‚Åª a ‚àßn nnf‚Åª b       -- ¬¨(a‚à®b) ‚â° ¬¨a ‚àß ¬¨b

to-nnf : Formula ‚Üí NNF
to-nnf = nnf‚Å∫
```

Reading:

- **`nnf‚Å∫ œÜ`** produces an NNF equivalent to **`œÜ`** (positive polarity).
- **`nnf‚Åª œÜ`** produces an NNF equivalent to **`¬¨ œÜ`** (negative
  polarity ‚Äî converting *the negation* of `œÜ`).

`nnf‚Åª` is not the negation of `nnf‚Å∫`'s result; it is "what `nnf‚Å∫` would
have produced for `¬¨f œÜ`, computed without ever building `¬¨f œÜ`
syntactically."

### Why this passes the termination checker

| Caller             | Argument            | Argument is a strict sub-term of input?       |
|--------------------|---------------------|-----------------------------------------------|
| `nnf‚Å∫ (¬¨f œÜ)`      | `nnf‚Åª œÜ`            | `œÜ` is a sub-term of `¬¨f œÜ`. yes              |
| `nnf‚Å∫ (a ‚àßf b)`    | `nnf‚Å∫ a`, `nnf‚Å∫ b`  | `a`, `b` are sub-terms of `a ‚àßf b`. yes       |
| `nnf‚Å∫ (a ‚à®f b)`    | `nnf‚Å∫ a`, `nnf‚Å∫ b`  | yes                                           |
| `nnf‚Åª (¬¨f œÜ)`      | `nnf‚Å∫ œÜ`            | yes                                           |
| `nnf‚Åª (a ‚àßf b)`    | `nnf‚Åª a`, `nnf‚Åª b`  | yes                                           |
| `nnf‚Åª (a ‚à®f b)`    | `nnf‚Åª a`, `nnf‚Åª b`  | yes                                           |

Every recursive call passes a constructor-stripped sub-formula. The
"flip" we used to do by *building syntax* (`¬¨f a ‚à®f ¬¨f b`) is now done by
*switching which function we call* (`nnf‚Åª a ‚à®n nnf‚Åª b`) ‚Äî at the meta
level instead of the object level.

Recurring Agda idiom: **when a logical rewrite would build new syntax
that breaks termination, encode the rewrite as a choice between mutually
recursive functions over the original syntax.**

### `to-nnf` is just a one-liner

```agda
to-nnf : Formula ‚Üí NNF
to-nnf = nnf‚Å∫
```

By specification `nnf‚Å∫ œÜ` is an NNF equivalent to `œÜ` ‚Äî exactly what
`to-nnf` should produce.

## 7. Soundness sketch

Not required as a formal proof, but here is why it is correct.

**Claim.** For every assignment `œÅ` and every formula `œÜ`,

```
eval œÅ œÜ                  ‚â° eval-nnf œÅ (nnf‚Å∫ œÜ)         (S‚Å∫)
eval œÅ (¬¨f œÜ)             ‚â° eval-nnf œÅ (nnf‚Åª œÜ)         (S‚Åª)
```

(Both sides use the `Maybe Bool` semantics from Problems 5 and 6, lines
147‚Äì177; equality is propositional equality of `Maybe Bool`.)

`(S‚Å∫)` and `(S‚Åª)` together imply `eval œÅ œÜ ‚â° eval-nnf œÅ (to-nnf œÜ)`.

Proof by **mutual structural induction on `œÜ`**, one case per equation
of `nnf‚Å∫`/`nnf‚Åª`.

### Case `œÜ = a ‚àßf b`, claim `(S‚Å∫)`

We need

```
eval œÅ (a ‚àßf b)  ‚â°  eval-nnf œÅ (nnf‚Å∫ a ‚àßn nnf‚Å∫ b)
```

Both sides lift `_and_` (with `nothing`-propagation) over their
respective evaluations. By IH `(S‚Å∫)` on `a` and `b`, the arguments are
equal, so the results are equal.

### Case `œÜ = a ‚àßf b`, claim `(S‚Åª)`

We need

```
eval œÅ (¬¨f (a ‚àßf b))  ‚â°  eval-nnf œÅ (nnf‚Åª a ‚à®n nnf‚Åª b)
```

LHS unfolds to `not` applied to `eval œÅ a and eval œÅ b`. RHS unfolds to
`eval-nnf œÅ (nnf‚Åª a) or eval-nnf œÅ (nnf‚Åª b)`. By IH `(S‚Åª)` on `a`, `b`,

```
eval-nnf œÅ (nnf‚Åª a) = eval œÅ (¬¨f a) = lift not (eval œÅ a)
eval-nnf œÅ (nnf‚Åª b) = eval œÅ (¬¨f b) = lift not (eval œÅ b)
```

So RHS is `(not x) or (not y)` ‚Äî which equals `not (x and y)` by De
Morgan on `Bool`. The `nothing` cases line up by inspection.

### Other cases

- **`var n`, `(S‚Å∫)`:** `eval œÅ (var n) = lookup n œÅ = eval-lit œÅ (pos n)
  = eval-nnf œÅ (lit (pos n))`. Direct.
- **`var n`, `(S‚Åª)`:** `eval œÅ (¬¨f (var n)) = lift not (lookup n œÅ) =
  eval-lit œÅ (neg n) = eval-nnf œÅ (lit (neg n))`. Direct.
- **`¬¨f œà`, `(S‚Å∫)`:** `eval œÅ (¬¨f œà) = eval-nnf œÅ (nnf‚Åª œà)` is `(S‚Åª)`
  for `œà` (the IH).
- **`¬¨f œà`, `(S‚Åª)`:** `eval œÅ (¬¨f (¬¨f œà)) = lift not (lift not (eval œÅ
  œà)) = eval œÅ œà = eval-nnf œÅ (nnf‚Å∫ œà)` by `(S‚Å∫)` on `œà`. Double
  negation.
- **`a ‚à®f b`:** symmetric to `‚àßf`, with `or`/`and` swapped.

Writing the proof in Agda is a worthwhile exercise (not very long modulo
the `Maybe`-noise) but is not part of the project.

## 8. Worked example

Let `p ‚âî var 0`, `q ‚âî var 1`, `r ‚âî var 2`. Consider

```
œÜ  =  ¬¨f ((p ‚à®f ¬¨f q) ‚àßf r)
```

Tracing `to-nnf œÜ = nnf‚Å∫ œÜ`:

```
nnf‚Å∫ (¬¨f ((p ‚à®f ¬¨f q) ‚àßf r))
=  nnf‚Åª ((p ‚à®f ¬¨f q) ‚àßf r)               -- nnf‚Å∫ on ¬¨f case
=  nnf‚Åª (p ‚à®f ¬¨f q) ‚à®n nnf‚Åª r            -- ¬¨(a‚àßb) ‚â° ¬¨a ‚à® ¬¨b
=  (nnf‚Åª p ‚àßn nnf‚Åª (¬¨f q)) ‚à®n nnf‚Åª r     -- ¬¨(a‚à®b) ‚â° ¬¨a ‚àß ¬¨b
=  (nnf‚Åª p ‚àßn nnf‚Å∫ q) ‚à®n nnf‚Åª r          -- ¬¨¬¨œÜ ‚â° œÜ
=  (lit (neg 0) ‚àßn lit (pos 1)) ‚à®n lit (neg 2)
```

Informally:

```
(¬¨p ‚àß q) ‚à® ¬¨r
```

By hand: `¬¨((p ‚à® ¬¨q) ‚àß r) ‚â° ¬¨(p ‚à® ¬¨q) ‚à® ¬¨r ‚â° (¬¨p ‚àß ¬¨¬¨q) ‚à® ¬¨r ‚â°
(¬¨p ‚àß q) ‚à® ¬¨r`. Each step matches one of the four rewrites in ¬ß5,
encoded as a function call rather than syntax rewrite.

As Agda values:

```agda
-- Original Formula
example-formula : Formula
example-formula = ¬¨f ((var 0 ‚à®f ¬¨f var 1) ‚àßf var 2)

-- Expected NNF
example-nnf : NNF
example-nnf = (lit (neg 0) ‚àßn lit (pos 1)) ‚à®n lit (neg 2)

-- They agree:
-- to-nnf example-formula  ‚â°  example-nnf
```

(Verify with `_ : to-nnf example-formula ‚â° example-nnf` and `_ = refl`;
`refl` works if the equality holds definitionally.)

## 9. NNF, DNF, CNF: the family

NNF is the common ancestor of two more restrictive normal forms.

- **Conjunctive Normal Form (CNF)** is an NNF that is a *conjunction* of
  *disjunctions* of literals ‚Äî equivalently, no `‚à®n` has an `‚àßn`
  descendant. Problem 7 (lines 191‚Äì198):

  ```agda
  data Disjunct : Set where
    lit  : Literal ‚Üí Disjunct
    _‚à®d_ : Literal ‚Üí Disjunct ‚Üí Disjunct

  data CNF : Set where
    dis  : Disjunct ‚Üí CNF
    _‚àßc_ : Disjunct ‚Üí CNF ‚Üí CNF
  ```

  Same trick as `NNF` vs `Formula`: bad shapes ruled out by construction.

- **Disjunctive Normal Form (DNF)** is the dual ‚Äî a *disjunction* of
  *conjunctions* of literals. Not defined in Agda for this project, but
  the obvious mirror.

The pipeline:

```
Formula  --to-nnf-->  NNF  ----->  CNF   (or DNF)
```

The second arrow is *not* free: naive distribution from NNF to CNF/DNF
can blow up exponentially. That motivates the **Tseytin transformation**
(Problem 10, note 06): an NNF to *equisatisfiable* (not equivalent ‚Äî
careful!) CNF whose size is linear in `|œÜ|`, via fresh variables for
internal nodes. Tseytin's input is exactly the `NNF` defined here.
